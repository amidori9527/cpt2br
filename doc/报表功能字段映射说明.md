## 1. 总体结构
根对象：

```typescript
BeeReportTemplate {
  reportReference
  reportConfig
}
```

结构可分为两层：

+ `reportReference`：报表底稿层

| 字段 | 描述 |
| --- | --- |
| source | 二维表格原始值 |
| meta | 单元格元信息 |
| merges | 单元格合并信息 |
| resized | 行列尺寸 |
| hidden | 行列隐藏 |
| floatElements | 浮动元素(自定义浮动内容、浮动图片、浮动图表) |


+ `reportConfig`：报表行为层

| engineConfig | 引擎配置 |
| --- | --- |
| eventConfig | 事件配置 |
| followUpPrintOpt | 续打 / 偏移打印配置 |
| footerRepeat | 页尾重复打印配置 |
| functionConfig | 报表函数配置 |
| headerFrozen | 冻结表头配置 |
| headerOptions | 页面头部工具栏配置 |
| headerRepeat | 页头重复配置 |
| pageConfig | 页面布局配置 |
| printOptions | 打印选项配置 |
| scopeConfig | 全局作用域变量配置 |
| searchBarConfig | 查询栏配置 |
| serviceConfig | 数据源配置 |
| splitLayout | 报表分栏配置 |

```javascript
  "row": 0,
  "col": 0
```

#### 2.1.1 单元格类型
单元格一共有五种类型，普通单元格、斜线单元格、子报表单元格、公式单元格、数据列单元格，用meta中的type字段来区分是哪种单元格。

+ 普通单元格：就是最基础的单元格，内容只有文本，在v里面存储单元格的文本内容。

字段配置示例如下：

```javascript
{
  row: 0,
  col: 0,
  v: "文字",
  t: 1
}
```

+ 斜线单元格：该单元格存放斜线的内容，斜线内容支持「普通」、「公式」两种类型，公式方式支持写入单元格地址、公式、函数等（动态配置），“diagonalType”存储的是斜线单元格的类型，diagonalValue存储的是斜线单元格的内容。

字段配置示例如下：

```javascript
//普通的斜线:
{
  row: 0,
  col: 0,
  diagonalType: "1" //普通
  diagonalValue: "病区|院区"
  type: "diagonal"
}
//公式的斜线:
{
  row: 0,
  col: 0,
  diagonalType: "2" //公式
  diagonalValue: "$$A1|$$sum($$scope.count,2)"
  type: "diagonal"
}

```

+ 子报表单元格：当前主报表中再嵌入其他报表，v存储的是「子报表cd+ “.report@”+ 子报表的版本号」，subReportConfig存储子报表的配置。

字段配置示例如下：

```javascript
{
  row: 0,
  col: 0,
  t: 1,
  v: "zbb.report@1.0.1",
  subReportConfig: {
    autoHeight: false,
    code: "zbb",
    integration: true,
    params: [{ //参数默认为空数组
      parameterDesc: "1",
      parameterName: "a",
      parameterScope: "",
      parameterType: "string"
    }], 
    version: "1.0.1"
  }
  type: "subReport"
}
```

+ 公式单元格：公式形态的单元格，可以使用函数，v存储公式内容。

字段配置示例如下：

```javascript
{
  row: 0,
  col: 0,
  t: 1,
  type: "formula",
  v: "$$seq()" //公式
}
```

+ 数据列单元格：放置数据集字段的单元格，v存储的是「“$DS.”+ 数据设置+ "(" +  数据集字段+")"」，field表示数据集字段，dsName表示是哪个数据集，dataSetting表示是数据设置类型（数据设置一共有三种，分别是分组、列表、汇总，汇总里面包含子类，分别是求和、平均、最大值、最小值、个数）。

字段配置示例如下：

```javascript
// 三种不同的数据类型，v跟dataSetting字段会有区别，汇总需要外加dataSettingDetail字段，表示汇总的子类
// dataSettingDetail: 0-求和、1-最大值 2-最小值、3-平均、4-个数。
// 分组
{
  row: 0,
  col: 0,
  t: 1,
  type: "columnData",
  v: "$DS.G(ds1.value)", 
  field: "ds1.value",
  dsName: "ds1",
  dataSetting: "group"
}
// 列表
{
  row: 0,
  col: 0,
  t: 1,
  type: "columnData",
  v: "$DS.S(ds1.value)", 
  field: "ds1.value",
  dsName: "ds1",
  dataSetting: "list"
}
// 汇总
{
  row: 0,
  col: 0,
  t: 1,
  type: "columnData",
  v: "$DS.求和(ds1.value)", 
  field: "ds1.value",
  dsName: "ds1",
  dataSetting: "group",
  dataSettingDetail: 0
}
```

#### 2.1.2 单元格形态
单元格形态分为图片形态、二维码形态、条形码形态、数据字典形态。在meta中通过contentType字段去区分是哪个形态。

+ 图片形态：将单元格的值以图片的形式展示，可以直接上传图片，或者放一个数据集字段。

```javascript
{
  row: 0,
  col: 0,
  t: 1,
  contentType: "imageShape",
  v: "upload/895096972380209200", 
  image： {
    fileId: "895096972380209200",
    mode: "auto"
  }
}
```

+ 二维码形态：将单元格的值以二维码的形式展示。

```javascript
{
  row: 0,
  col: 0,
  t: 1,
  contentType: "qrCode",
  v: "Y123", 
  qrcodeOptions： {
    colorDark: "#000000",
    colorLight: "#ffffff",
    correctLevel: "H",
    height: 90,
    margin: 10
  }
}
```

+ 条形码形态：将单元格的值以条形码的形式展示。

```javascript
{
  row: 0,
  col: 0,
  t: 1,
  contentType: "barCode",
  v: "Y123", 
  barcodeOptions： {
    displayValue: true,
    format: "CODE39",
    width: 2,
    height: 30,
    margin: 10
  }
}
```

+ 数据字典形态：通过数据字典将原始数据进行转换，支持「数据查询」、「自定义」两种类型。

```javascript
// 数据查询
{
  row: 0,
  col: 0,
  t: 1,
  type: "columnData",
  dsName: "ds1",
  v: "$DS.G(ds1.cate)", 
  field: "ds1.cate",
  dataSetting: "group",
  contentType: "dataDic",
  dataDicOptions： {
    type: "dataQuery",
    config: {
      actualColumnName: "cate",
      dataSet: "ds1",
      showColumnName: "xilie"
    }
  }
}
// 自定义
{
  row: 0,
  col: 0,
  t: 1,
  type: "columnData",
  dsName: "ds1",
  v: "$DS.G(ds1.cate)", 
  field: "ds1.cate",
  dataSetting: "group",
  contentType: "dataDic",
  dataDicOptions： {
    type: "custom",
    config: {
      valueMap: [{
        actualValue: "实际值1",
        showValue: "显示值1"
      }]
    }
  }
}
```

#### 2.1.3 单元格过滤
对单元格的内容进行过滤，支持「普通」、「公式」两种类型，类型通过filterType字段作区分，多个过滤条件可以通过「and」或「or」去合并，通过refLogic字段去表示。

普通的过滤配置中，操作符分为等于(=)，不等于(!=)，大于(>)，小于(<)，大于等于(>=)，小于等于(<=)，包含({)，不包含(!{)，值的类型有字符串、整型、双精度型、数据列、单元格、公式。

```javascript
// 普通
// 值的类型不同，需要存储的字段也不同（值的类型通过refType做区分）
// 字符串：refValue  整型：refValue  双精度型：refValue  数据列：refDSName、refField
// 单元格：refCell  公式：refValue
{
  row: 0,0.1
  col: 0,
  t: 1,
  type: "columnData",
  dsName: "ds1",
  v: "$DS.G(ds1.cate)", 
  field: "ds1.cate",
  dataSetting: "group",
  filter: [{ // 字符串
    field: "cate",
    filterType: 0,
    operator: "=",
    refCell: "",
    refDSName: "",
    refField: "",
    refLogic: "and",
    refType: "string",
    refValue: "分类1"
  }, { // 整型
    field: "cate",
    filterType: 0,
    operator: "=",
    refCell: "",
    refDSName: "",
    refField: "",
    refLogic: "and",
    refType: "int",
    refValue: 0
  }, { // 双精度型
    field: "cate",
    filterType: 0,
    operator: "=",
    refCell: "",
    refDSName: "",
    refField: "",
    refLogic: "and",
    refType: "double",
    refValue: "0.1"
  }, { // 数据列
    field: "cate",
    filterType: 0,
    operator: "=",
    refCell: "",
    refDSName: "ds1",
    refField: "xilie",
    refLogic: "and",
    refType: "columnData",
    refValue: ""
  }, { // 单元格
    field: "cate",
    filterType: 0,
    operator: "=",
    refCell: "0/0",
    refDSName: "",
    refField: "",
    refLogic: "and",
    refType: "cell",
    refValue: ""
  }, { // 公式
    field: "cate",
    filterType: 0,
    operator: "=",
    refCell: "",
    refDSName: "",
    refField: "",
    refLogic: "and",
    refType: "formula",
    refValue: "$$sum(B1)"
  }]
}
// 公式
//如果涉及单元格需要多一个字段：refCell
{
  row: 0,
  col: 0,
  t: 1,
  type: "columnData",
  dsName: "ds1",
  v: "$DS.G(ds1.cate)", 
  field: "ds1.cate",
  dataSetting: "group",
  filter: [{ // 过滤脚本中没有涉及单元格
    filterType： "1",
    refLogic: "and",
    filterScript: "$$year()"
  }, { // 过滤脚本中涉及单元格-需要使用refCell存储依赖的单元格位置
    filterType： "1",
    refLogic: "and",
    filterScript: "$$sum(B1)",
    refCell: ["0/1"]
  }]
}
```

#### 2.1.4 单元格数据设置
单元格数据设置有「分组」、「列表」、「汇总」三种，汇总再分为小类，分别是「求和」、「平均」、「最大值」、「最小值」、「个数」五种。分组跟列表通过跟单元格扩展（expandDirection字段）配合使用。

```javascript
// 分组
{
  row: 0,
  col: 0,
  t: 1,
  type: "columnData",
  dsName: "ds1",
  v: "$DS.G(ds1.cate)", 
  field: "ds1.cate",
  dataSetting: "group",
  expandDirection: "v" // 不是必要的，只是一般都配合使用
}
// 列表
{
  row: 0,
  col: 0,
  t: 1,
  type: "columnData",
  dsName: "ds1",
  v: "$DS.S(ds1.cate)", 
  field: "ds1.cate",
  dataSetting: "list",
  expandDirection: "v" // 不是必要的，只是一般都配合使用
}
// 汇总
// 汇总中还有小分类，存储在dataSettingDetail中，分别是：0-求和、1-最大值 2-最小值、3-平均、4-个数。
{
  row: 0,
  col: 0,
  t: 1,
  type: "columnData",
  dsName: "ds1",
  v: "$DS.求和(ds1.cate)", 
  field: "ds1.cate",
  dataSetting: "compute",
  dataSettingDetail: 0
}
```

#### 2.1.5 单元格条件属性
给单元格设置条件属性，expressionStatement字段是条件属性的表达式，可以返回true/false，也可以返回一个对象，如果返回对象，返回的对象里包含的属性会覆盖上方配置的属性。

<font style="color:#DF2A3F;">注：如果expressionStatement返回的对象里的属性，attributeActiveInfo中没有配置，则不会生效，所以attributeActiveInfo跟expressionStatement要配合使用。</font>

#### 2.1.6 单元格控件

**pc端**

+ 按钮

```javascript
{
  type: "button",
  scopeField: "",
  identifier: "Iypz", // 随机生成的标识符
  attribute: {
    buttonType: "normal",
    plain: true,
    size: "small",
    slot: "",
    type: "default"
  },
  componentLogic: {
    dataSource: null,
    events: null,
    insertCell: {
      rowend: 1,
      rowstart: 1,
      scopeField: ""
    },
    interaction: null,
    validate: null
  }
}
```

+ 选择器

```javascript
{
```
