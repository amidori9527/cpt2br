# CPT 转 BeeReport 工具用户操作文档

## 1. 工具说明

本工具用于将帆软 CPT 报表模板转换为 BeeReport 可导入的报表文件。

用户只需要选择 CPT 文件或 CPT 所在文件夹，按提示填写报表名称和报表标识，工具会自动完成转换并生成 zip 文件。

## 2. 启动工具

打开转换工具后，会看到提示：

```text
请输入 CPT 文件路径或文件夹路径：
```

此时可以输入：

- 一个 CPT 文件路径
- 一个包含 CPT 文件的文件夹路径

路径可以直接从文件管理器复制。

如果路径中包含空格，可以用引号包起来。

## 3. 转换单个 CPT 文件

如果输入的是单个 CPT 文件路径，例如：

```text
D:\报表模板\other.cpt
```

工具会继续提示：

```text
请输入转换后的报表名 na（默认：other）：
请输入标识符名 cd：
```

填写说明：

- `na`：报表名称，显示在 BeeReport 中。
- `na` 可以直接回车，默认使用 CPT 文件名。
- `cd`：报表标识符，不能为空，建议使用英文、数字或下划线。

转换完成后，会在原 CPT 文件所在目录生成一个 zip 文件：

```text
{cd}-1.0.1.zip
```

该 zip 文件就是导入 BeeReport 使用的文件。

## 4. 转换文件夹中的多个 CPT 文件

如果输入的是文件夹路径，例如：

```text
D:\报表模板
```

工具会查找该文件夹下的 CPT 文件，并按顺序逐个转换。

每个 CPT 文件都会单独询问：

```text
正在转换第 1/3 个：xxx.cpt
请输入转换后的报表名 na（默认：xxx）：
请输入标识符名 cd：
```

需要为每个 CPT 分别填写：

- 报表名称 `na`
- 报表标识符 `cd`

所有 CPT 转换完成后，只会生成一个 zip 文件：

```text
{文件夹名}-1.0.1.zip
```

这个 zip 中包含该文件夹下所有转换后的报表。

## 5. 转换完成后的信息

转换成功后，工具会显示类似内容：

```text
已生成：D:\报表模板\xxx-1.0.1.zip
包内文件：xxx-1.0.1.report
报表 ID：xxxxxxxxxxxxxxxxxxxxxxxx
表格尺寸：11x15
单元格 meta：114，合并区域：31
```

看到“已生成”表示转换成功。

需要导入 BeeReport 的文件就是提示中的 zip 文件。

## 6. 注意事项

1. 文件路径必须真实存在。
2. 单文件转换时，输入的文件必须是 `.cpt` 或 `.CPT`。
3. 文件夹转换时，只会转换该文件夹第一层的 CPT 文件，不会自动转换子文件夹里的文件。
4. 多个 CPT 转换到同一个 zip 时，`cd` 不能重复。
5. 如果不确定 `na` 填什么，可以直接回车使用默认文件名。
6. 建议 `cd` 使用简单英文或数字，避免特殊字符。

## 7. 常见问题

### 7.1 提示“路径不存在”

说明输入的文件或文件夹路径不正确。

请重新复制路径后再输入。

### 7.2 提示“文件不是 .cpt”

说明输入的是非 CPT 文件。

请确认选择的是帆软 `.cpt` 模板文件。

### 7.3 文件夹里有 CPT，但没有被转换

工具只会读取当前文件夹第一层的 CPT 文件。

如果 CPT 在子文件夹中，请直接输入该子文件夹路径。

### 7.4 提示 cd 重复

同一个 zip 中不能有相同的报表标识符。

请为当前报表重新输入一个不同的 `cd`。

### 7.5 转换后的报表和帆软不完全一致

BeeReport 当前只支持核心报表功能。

帆软中部分高级配置如果 BeeReport 暂未支持，转换时会自动跳过。
